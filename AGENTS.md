Guardrails for AI agents and contributors in the HearthShelf mobile app.

## What this repo is

The HearthShelf **mobile app** - Expo / React Native, TypeScript. It signs into
the hosted front door (`app.hearthshelf.com`) via Clerk, connects to the user's
linked ABS server, and plays their library - including in the car via **Android
Auto** (a native Kotlin Media3 `MediaLibraryService`).

- **Auth:** Clerk (Google OAuth) -> control-plane grant -> `/hs/hosted/connect`
  -> per-user ABS token. Mirrors `HearthShelf-WebApp`'s flow.
- **Audio:** `react-native-video` (Media3 engine; background + lock-screen).
- **Android Auto:** native Media3 `MediaLibraryService` in
  `plugins/hearthshelf-auto` (an Expo config plugin, so it survives `prebuild`).
  Google forbids the Car App Library *template* model for car audio - a media
  app MUST use a MediaBrowserService. Don't reintroduce a template-based lib.
- **Shared code:** ABS types + pure logic come from `@hearthshelf/core`, wired
  as a git submodule at `packages/core`. Clone with `--recursive` (or
  `git submodule update --init`). Don't re-add a local copy of the ABS types.

## Hard rules

- **TypeScript strict, no `any`.** Run `tsc --noEmit` before committing.
- **The car surface is native and headless-capable.** It fetches ABS directly
  using the token JS hands it (`HearthShelfAuto` native module -> SharedPrefs).
  Keep the browse tree + play-resolution logic in the Kotlin service; the JS
  side only hands over the server URL + token at sign-in.
- **`android/` and `ios/` are gitignored** (regenerated by `expo prebuild`). The
  Android Auto native source of truth is `plugins/hearthshelf-auto/` - edit
  there, not in the generated `android/`.
- **Never reference example/other apps** in user-facing text or commit messages.
- **Don't push** unless asked. Commit directly to `main` is fine pre-release.

## Licensing

**MIT** (see `LICENSE.md`) - deliberately permissive, not AGPL like the servers.
AGPL on a client app buys little (its network-copyleft clause doesn't apply to an
app running on the user's phone) and costs a real Apple App Store conflict
(GPL/AGPL vs App Store terms - the VLC issue) plus contributor deterrence. MIT
sidesteps both. The shared `@hearthshelf/core` is MIT too, so it links cleanly.
The paid iPhone build recoups Apple's developer fee by selling **convenience**
(the signed App Store build), not by closing the source - keep the source open.
