Reconciliation · v2 + v3 → FINAL

What the two reviews agreed on, and where they were wrong

This FINAL page keeps the consultant layout (stable section order, one-tap visible Finish, accent Resume, the fixed BookActionsSheet grid) and folds in the one real gap both reviews flagged: the ebook reader had no way to move to an arbitrary spot. Below is the ledger of rejected and adopted calls.

Rejected from the reviews

  • v2's move-Finish-to-overflow. Finish stays a visible one-tap square — v3 agreed. Hiding the most common completion action is worse, not safer.
  • v2's "Resume · Ch 12" label. The chapter number can disagree with a prologue or a named chapter — v3's catch. The primary button reads plain Resume / Start / Listen again.
  • v2's "reader has no light/sepia theme" claim. INVALID — prod read.tsx:244 already ships Light / Sepia / Paper / Dark. The real gap was the position slider, now added.

Adopted from the reviews

  • Ebook position slider + "min left" + scope pill. Both reviews' top pick, and a real prod gap — read.tsx:217 only has ± chapter chevrons and a percent readout.
  • Accessibility + reduce-motion floor. Both reviews. Reduce Motion turns EmberBurst static; every icon-only control gets a screen-reader label; text reflows for Dynamic Type.
Screen 04a · Book detail

Book detail

The hub for one book: hero, progress, primary actions, chapters, series, about, community, notes. The core redesign idea is spatial stability — every section lives in the same place in every listening state, so the second time you open any book detail, your thumb already knows where Finish, Download, and Chapters are.

Teardown — what's wrong today

  • Sections reorder by listening state. In-progress puts status→CTA→listening-now→series→chapters; finished pushes chapters last; not-started puts About second. Three different pages for one screen — zero muscle memory (D-CONSIST).
  • Bookmarks entry only exists when count > 0. The empty-state copy "Add them from the player" is written in code but unreachable — the icon that would open it is hidden exactly when you'd need the hint (D-ACTIONS, D-STATES).
  • Every finish pays the date interstitial. The Finish square opens the mandatory "When did you finish?" prompt even for "just now" — the 95% case pays for the 5% backdating case (D-FINISH).
  • Download square hides its cost. No file size before you commit; while downloading, cancel is the same square relabeled "{pct}%" with no visible cancel affordance.
  • Reset progress and Hide don't exist here. They live only behind long-press on specific shelves — the book's own page can't do everything the long-press sheet can (D-ACTIONS).
  • Star rating row vanishes at 0 and pushes everything below it up — another source of layout shift between books.

Redesign moves

  • One stable order for all states: Hero → Status + CTA → Chapters → Series → About → Community → Notes → Club. Only the CTA label adapts (Start / Resume / Listen again). Muscle memory is the point — Finish is always the first square, Download always the last.
  • Finish is one tap — EmberBurst on the square, actionable toast "Finished · Edit date / Undo". The date prompt survives, reached only from the toast or bulk flows (D-FINISH). Finish stays a visible one-tap action — we reject moving it to overflow; hiding the most common completion action is worse, not safer.
  • Primary button reads plain Resume / Start / Listen again — never "Resume · Ch 12". The chapter number can disagree with a prologue or a named chapter, so the label stays a verb, not a location.
  • Accessibility + reduce motion (D-A11Y): Reduce Motion turns EmberBurst into a single static confetti frame and the entrance stagger into one fade; every icon-only control (bookmarks, overflow, back) carries a screen-reader label; text reflows for Dynamic Type up to ~200% without clipping (hero title clamps then truncates, action tiles grow vertically).
  • Download square earns trust: shows "1.2 GB" before you tap; while downloading it becomes a progress ring with an explicit ✕ inside; downloaded state keeps remove-with-confirm (tri-state kept).
  • Bookmarks icon always visible in the header — badge when count > 0, designed empty state with a player shortcut when zero (D-ACTIONS, D-STATES).
  • Overflow carries the full grammar: Share, Add to list, Recent Listens kept; Reset progress and Hide added; file-info footer kept (D-ACTIONS).
  • Kept: CoverGlow, cover-tap lightbox, tappable author & narrator, chapters window-of-4 with View all, series card with next-in-series eyebrow, About 6-line clamp, finished-by / listening-now avatar chips, self-hiding empty sections, notes row, sessions sheet.
  • Tab bar stays lit for the tab you came from — Library here (D-NAV).

Screen 04a · Book detail — sheets

Overflow & bookmarks sheets

The detail page's two header sheets. Overflow becomes the full action set for this book — nothing the long-press sheet can do is missing here anymore. Bookmarks gets the empty state its copy always deserved.

Teardown — what's wrong today

  • Overflow is missing half the grammar. Share, Add to list, Recent Listens, file info — but no Reset progress and no Hide; those exist only via long-press on certain shelves (D-ACTIONS).
  • The bookmarks empty state is dead code. "Add them from the player" is written but the entry point hides at count 0, so no user has ever read it (D-STATES).
  • Sessions sheet is solid (device icon, date, duration, tap-to-jump) but only discoverable through overflow — fine, kept as-is.

Redesign moves

  • Overflow = full set: Share, Add to list, Recent Listens kept; Reset progress (confirm) and Hide (undo toast) added; codec · bitrate · size footer kept (D-ACTIONS).
  • Bookmarks sheet, empty state designed: icon + one line + "Open player" shortcut CTA (D-STATES). Populated rows keep tap-to-play and optimistic ✕ delete.
  • Destructive rows confirm in place — the row morphs to "Reset · Cancel" instead of stacking another dialog.

Screen 04a · Book detail — states

  • Skeleton mirrors the stable order — cover, title lines, CTA row, chapter rows shimmer in the exact slots content will fill. No reflow on load.

Screen 04b · Ebook reader

Ebook reader

The reading surface tints its chrome to the page palette — that stays. What it's missing is any way to move through the book except one page at a time or the TOC. The redesign gives it the audio player's scrubbing metaphor: a chapter-scoped position slider with a book-wide toggle (D-CONSIST).

Teardown — what's wrong today

  • No position slider. The bottom bar is ◀ {pct}% · chapter ▶ — moving 50 pages costs 50 taps or a TOC detour (D-THUMB).
  • Tap zones are undiscoverable. Nothing hints that the page edges turn pages or that anything toggles the chrome.
  • No sense of pace — percent only; no "how long is left in this chapter" while reading.
  • Error screen prints the raw exception with only Go back (fixed in the states board above).

Redesign moves

  • Position slider fixes the top gap: a real draggable track with a knob, chapter-scoped with a Book toggle — the exact metaphor of the audio scrubber, so the two mediums scrub the same way (D-CONSIST). This is the fix for having no arbitrary-position navigation in prod (read.tsx only has ± chapter chevrons + a percent readout). Page arrows kept beside it for single-step precision.
  • Standard tap zones: left third = page back, right third = page forward, center = toggle chrome. First-open coach overlay shows them once (new).
  • "≈ 9 min left in chapter" estimate from the user's measured reading pace (new).
  • Kept: page-palette-tinted chrome, CFI position persisted per book, theme swatches (Light / Sepia / Paper / Dark), typeface Serif / Sans / Dyslexic, size stepper, Scroll / Paged layout, TOC panel — both panels reframed as bottom sheets.

Screen 04c · Upcoming release

Upcoming & pre-order

The page for a book that doesn't exist yet. It sells anticipation — so the countdown has to actually count, and the page should feel like a book detail, not a stripped-down cousin of one.

Teardown — what's wrong today

  • The countdown never ticks. "{X days} until release" is computed once on mount — leave the screen open over midnight and it lies (shipped bug).
  • No CoverGlow — the one screen selling a cover renders it flat, unlike every book detail (D-CONSIST).
  • No series context. A Book 6 pre-order doesn't link to the five books you own.
  • "Releases {date}" with no timezone clarity — midnight where?

Redesign moves

  • Live countdown: re-derive every 60 s (every 1 s under one hour); at zero it flips to "Out today" with a gold chip — specified in the motion table.
  • CoverGlow added — same bloom as book detail (D-CONSIST).
  • Series context row linking to the series screen, with owned-count (new).
  • "in your timezone" caption under the release date (new).
  • Kept: Follow toggle (bell → "We'll notify you", optimistic with rollback), View on Audible, description, cover-tap lightbox, back-only header.

Screen 04d · Shared action surfaces

The action grammar: sheets, selection, prompts, lightbox

Five surfaces every book screen shares. The rule (D-ACTIONS): one BookActionsSheet, one row order, everywhere books appear — no more actions that exist on one shelf and not another. Finish flows through the toast; the date prompt becomes an option; the lightbox stays exactly as shipped.

Teardown — what's wrong today

  • The sheet's contents depend on where you long-pressed. Reset progress appears only from the listening shelf; there's no Play, no queue actions, no View series — the sheet can't do what the tile it covers can (D-ACTIONS).
  • Mark finished in the sheet gates the date prompt, same as everywhere (D-FINISH).
  • Multi-select toolbar is icons-only plus an accent count — no "N selected" words, and rare ops crowd the same bar as common ones.
  • FinishDatePrompt is a toll booth — a well-built one (month chips, 15-year scroll, Now, clamped to today) charged on every single finish.
  • NotOwnedSheet and the lightbox are good. Three-phase request flow with busy + approval-aware success; pinch 1–4×, double-tap, pan, scrim-tap dismiss — keep both.

Redesign moves

  • Grid + pinned rows, all surfaces: the six peer launches (Play · Play next · Add to queue · Add to list · Download · Finish) are a 3-across icon grid — matching the player's More tray, so the app's two most-used action sheets share one launcher language. The stateful / destructive / drill-in items (View series · Share · Reset progress · Not right now) stay pinned rows below a divider — carrying their context labels and destructive color, and deliberately out of one-tap-by-accident range. Items self-hide only when truly inapplicable (series row hides for standalone books).
  • Every destructive or reversible action lands as an Undo toast — hide, reset, unfinish, remove download: same pattern, same position (D-FINISH pattern generalized).
  • Multi-select: icons keep their bar, gain 11px labels + "N selected" text; rare ops (hide, reset) move behind ⋯ (D-CONSIST).
  • FinishDatePrompt reframed: reached only from the finish toast's "Edit date" or bulk backdating. Identical internals — chips, Now, clamp, swipe-to-cancel.
  • NotOwnedSheet + Lightbox kept as-is, annotated below so the spec is complete.
