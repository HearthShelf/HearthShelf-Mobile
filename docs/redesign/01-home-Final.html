What changed after review

Rejected from the reviews

  • v2's status-line + avatar-merge + pin-on-scroll resume bar — over-engineered, and built on a "hero-hijack" regression v2 itself later admitted was invented (prod's hero is already volatile and has no shelf quick-play to break).
  • v3's "keep the beta Home as-is" — too conservative; the 340px hero, missing states, and 2–3-tap play paths are real problems the redesign fixes.

Adopted from the reviews

  • v3's header utility row — the header earns its keep: server/library context beside the greeting, plus Downloads (with an activity dot while anything is in-flight) and Search, all 44px targets.
  • A11y + reduce-motion floor (both reviews) — Reduce Motion turns the CoverGlow breathe into a static wash, the shelf stagger into one 120ms fade, and EmberBurst into a single frame; hero and tiles reflow for Dynamic Type without clipping; every icon-only control is labelled.
Screen 00 · App shell

Tab bar & mini dock

The navigation chrome every screen inherits. Today it forgets where you are the moment you drill into anything, and re-tapping a tab does nothing. The redesign makes the shell an always-honest map.

Teardown — what's wrong today

  • No active tab on any pushed route — item, series, group, search and club screens all re-render the bar with activeName=null. You lose "you are here" on every drilldown.
  • Re-tapping the active tab does nothing — no scroll-to-top on Home or Library, a near-universal tab-bar expectation.
  • 10px tab labels under a 30px visual pill — below the legibility floor.
  • Three competing player entry points (Now tab, mini dock, Home hero) with no rule for which appears when.
  • Mini dock offsets and the A-Z rail bottom inset are coupled by magic numbers — chrome pieces don't know about each other.

Redesign moves

  • Owning tab stays lit on pushed routes (D-NAV) — a book opened from Library keeps Library lit; from Home keeps Home lit.
  • Re-tap active tab = scroll to top; second re-tap on Library focuses search (D-NAV).
  • Labels raised to 11px, pill to 34px; whole 74px cell remains the touch target.
  • Entry-point rule: the mini dock is the player's ambassador everywhere except surfaces that already embed the player (Home hero while playing, Now tab, /player). One player, one ambassador per screen.
  • Toast band, dock, and rail share one layout constant — chrome stacks predictably: toast above dock, dock above tabs.

Screen 01 · Home

The hearth, rebalanced

Home's job: get you back into your book in one tap, then show you what's next. Today a ~340px hero spends the whole first viewport on one book and pushes everything else below the fold. The redesign keeps the spotlight but compresses it to ~240px, adds a glanceable dashboard row, and gives every shelf tile one-tap play.

Teardown — what's wrong today

  • ~340px hero for one book — stats, clubs and shelves start below the fold on most phones.
  • Playing any non-hero book costs 2+ taps — tile → detail → play. No quick-play anywhere (D-PLAY).
  • "See all" opens an 85% sheet holding a full grid — inconsistent with Library, awkward to browse, and back-gesture-hostile.
  • No empty, no error state — a failed load and an empty library look identical: a lonely greeting.
  • Shelf icons are guessed by string-matching the label — custom recommendation rows get mislabeled.
  • Club shelf pops in after fetch with no placeholder — visible layout shift.
  • Tiles carry no progress or finished status — only a download badge.
  • Release banner surfaces only the soonest book; "+N more" is dead text with no destination.

Redesign moves

  • Compact hearth hero (~240px) — blurred-art band, title, chapter, progress and transport in the thumb-friendly lower half. Whole band taps into the player.
  • Dashboard row under the hero: Up-next queue peek and streak/week tile — the user's own dashboard-grid exploration, fused with the hero rather than replacing it.
  • Quick-play chips + progress bars on tiles (D-PLAY) — shelf books start in one tap; long-press still opens actions.
  • "See all" pushes a real screen — same grid engine as Library, back gesture works, Home stays lit (D-NAV).
  • First-run, error and offline states designed (D-STATES).
  • Shelves declare their own icon in shelf metadata — no string guessing.
  • Club shelf reserves its height with skeleton cards until loaded.
  • Release banner taps through to the book; long-press or the count chip opens the full Following list.
  • Header utility row (FINAL, from v3) — server/library context + Downloads + Search live in the hero bleed; the header earns its keep.
  • Reduce Motion + Dynamic Type (FINAL, both reviews) — static glow wash, single-fade shelves, one-frame EmberBurst; hero clamps to 2 lines then truncates at large text; tiles grow vertically, art stays fixed.
