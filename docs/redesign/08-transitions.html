Motion 08 · Screen transitions

What's wrong today, and the three ways to fix it

Every push/pop and tab switch should feel like the same app moving. Today it doesn't: transitions are whatever the platform default happens to be, per screen, per OS.

Teardown — what's wrong today

  • No declared animation on any pushed route — app/_layout.tsx sets an animation only for /player; book, series, club, search, shelf, group all inherit the platform default, so iOS parallax-slides while Android does its own thing (and changes between OS versions).
  • Tab switches are a hard cut — swapping Home ⇄ Library ⇄ Stats teleports with zero motion, which reads as a glitch next to the animated player.
  • Player still swipe-dismisses — the collapse chevron was removed, but gestureEnabled: false only kills the iOS edge-pop gesture. The iOS full-screen sheet swipe and Android's system back gesture still pop the route, sliding the player away. Needs a code fix regardless of which option wins (block the dismiss gesture properly; back button should collapse to the mini player, not "exit" the primary surface).
  • Two unrelated motion vocabularies — player rises from the dock (vertical), pages slide sideways (horizontal), tabs cut (none). Three grammars, no system.

Redesign moves

  • Declare one animation for every route — set it once on the root Stack.screenOptions so iOS and Android are pixel-identical.
  • Give tab swaps a micro-transition — 120–200ms of fade is enough to make switching feel intentional, never sluggish.
  • Pick one grammar — the three options below are each a complete, self-consistent system (push + pop + tab swap). Pick one; don't mix.
  • Respect reduced motion — all options collapse to a plain fade when the OS accessibility setting asks for it.

Motion 08 · Foldables

How each system survives the open fold

The fold direction (docs/redesign/fold/) changes what a "page swap" even is: on the open device the left leaf keeps focus, selection, and playback while most navigation becomes a right-leaf pane swap — and the hinge plus 24px of gutter on each side must stay motion- and touch-free. A transition system only qualifies if the exact same recipe works whole-screen when folded and per-pane when open.

What the fold rules eliminate

  • Option A is out. A horizontal slide drags content across the hinge, and in dual-pane the parent stays visible in the left leaf — there is nothing to "slide over". Worse, the horizontal axis is already the spine's spatial model (right leaf = detail of the left leaf), so sideways motion inside a pane fights it. A would need a second grammar for the open posture — the exact inconsistency this page exists to kill.
  • Whole-screen pushes are the exception, not the rule, when open — only compact-posture navigation and the player's rise stay full-screen. Any system defined only at screen level doesn't compose.

What passes

  • Option B passes clean — directionless fade-through scopes to a single leaf with zero changes; it's literally the reduced-motion rule the fold prototype already specifies for right-pane swaps.
  • Option C passes with one posture rule — the rise runs parallel to the vertical hinge in book posture, so motion never crosses the seam. In tabletop posture the fold is horizontal, so the rise is dropped there and C degrades to a pure fade (tabletop is the player surface anyway, which doesn't push routes).
  • C and B converge — C's reduced-motion and tabletop fallback is B. Choosing C gives both: lift where the axis allows it, calm fade where it doesn't.
Open fold — right-leaf pane swap option C, same keyframes
Library stays put in the left leaf; selecting a book swaps only the right leaf, which loops the identical Shelf Lift keyframes the phone demo uses. Nothing moves across the seam; the 24px gutters stay quiet.

Cross-review — study 09 (Steady Frame / Shelf Depth / Cover Relay)

  • Adopted from 09: entry motion is a promise. This page originally kept the player's big slide-from-bottom rise as the "hero variant" of Option C. Study 09's Rule 03 is right that this is sheet grammar — a full-screen bottom slide teaches users to swipe down to dismiss, which is exactly the gesture we're removing. Amendment to C: the player enters with the same standard Shelf Lift as every other push (or, later, as a morph growing out of the mini-player dock). One grammar, now with zero exceptions.
  • Adopted from 09: refuse the swipe audibly. A blocked gesture must not feel dead. On a downward drag attempt, the player follows the finger a few px, rubber-bands back (~260ms spring), and a toast says "Player stays open". Ship this with the gesture fix.
  • Held against 09: foldables. Study 09 never tests the hinge. Its "Steady Frame" pick soft-pushes details sideways (18% travel) — pane-clippable, so survivable, but it reintroduces the horizontal axis the fold analysis eliminated; its "Cover Relay" self-admits the highest foldable/virtualization QA cost. One dose-scaled grammar (C) composes across postures better than three route-type grammars.
  • Held against 09: doctrine. 09's demo swaps the Stats tab for a Search tab — against D-NAV/D-SEARCH (search is a pushed route, not a tab). Cosmetic in a motion study, flagged so it doesn't leak into code.
  • Parked: Cover Relay. The book-cover shared-element morph is the most "HearthShelf" idea in either study, and it's orthogonal — it can be layered on top of C later for book → detail → player, once the base system ships. Not in scope for the first pass.

Recommendation

  • Option C — Shelf Lift, as amended by the 09 cross-review. One dose-scaled grammar for everything — pushes, pops, tabs, and now the player too (standard lift instead of the old sheet-like full slide, so the entrance no longer promises a swipe-down that doesn't exist). It's a one-line native animation on both platforms (no custom animator to maintain) and fold-proof: the lift runs parallel to the book-posture hinge and scopes to a single leaf unchanged. Option B is the fallback (it's what C already degrades to under reduced motion and in tabletop posture). Option A is eliminated by the fold direction — horizontal slides cross the hinge and fight the spine's left/right spatial model, forcing a second grammar for the open posture. Study 09's "Cover Relay" morph is parked as a later layer on top of C.
  • Ship the player-gesture fix with whichever wins: fully block swipe-dismiss on /player (the current gestureEnabled:false misses the iOS full-screen swipe and Android system back), make hardware/system back collapse to the mini player rather than pop, and refuse drag attempts audibly — follow the finger a few px, rubber-band back, toast "Player stays open" (adopted from study 09).
