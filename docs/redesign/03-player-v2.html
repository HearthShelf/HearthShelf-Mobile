The consultant fixed two real shipped bugs (disappearing skip buttons, the fake "car mode coming soon" stub) and moved the queue into the thumb zone. All correct. But two of the redesign's decisions remove behavior users already rely on. This is where a second opinion earns its keep.

reviewsScored against the consultant's 03-player.html

The player is the screen users hold longest, so every change compounds. One decision is a clear regression; one was overstated.

check_circleKept — ship as written

  • Skip buttons unconditional (chapterless bug fix — verified real at player.tsx:627). Correct and overdue.
  • Queue → "Up next" in the thumb zone, sync stays a header status. Right split.
  • Car mode unification — one concept, kills the coming-soon lie. Yes.
  • Buffering ring + no redundant chapter pill. Good restraint.

constructionStill wrong — unaddressed

warningRegressed — the redesign made it worse

Screen 03 · Player

Car mode (was "immersive")

Today the swipe-up "immersive" layout IS the car layout — big cover, centered transport, extra chapter row — yet the action row ships a separate "Car mode" button that toasts "coming soon". One feature, two names, one of them fake. The redesign merges them: one concept, one name, two entrances.

Teardown — what's wrong today

  • Duplicate concept — immersive mode (swipe up, working) and the carMode action button ("coming soon" toast) are the same feature; users who tap the button are told it doesn't exist while it's one gesture away (D-CONSIST).
  • Exit is gesture-only — swipe down works, but at arm's length in a dashboard mount an undiscoverable gesture is the wrong only-exit.
  • Good bones — hidden chrome, bigger cover, centered transport and the second chapter row are already right for glanced-at use.

Redesign moves

  • One name: Car mode — entered by swipe-up OR the carMode action button, which now actually works (kills the stub). Exit by swipe-down or the big X (D-CONSIST).
  • Glance-first sizing — 120px play, 88px skips, 34px title, 40px scrubber. Everything hits at arm's length without looking twice.
  • Chapter row promoted — prev · chapter name · next as a full-width row above the transport; the name is the tap target for the chapters sheet.
  • Sleep + speed stay reachable — two oversized pills at the bottom; everything else waits in the phone layout.
Carousel — up-next page NO CHANGES

Sheets I — chapters, speed, sleep

The listening-control sheets. Chapters can't find "now" in a 50-chapter book; sleep's preset pills look armed but aren't — the single worst trap in the app (users walk away thinking the timer runs). Speed is already good and mostly keeps.

Teardown — what's wrong today

  • Chapters sheet opens at the top — no scroll-to-current, no search; finding chapter 41 of 60 is a scroll safari. The row anatomy (now-playing icon / green check / index + title + duration, tap = seek + dismiss) is fine.
  • Sleep presets don't arm — tapping "30 m" only moves the slider; without an explicit Start nothing happens. Shipped bug per the systemic teardown (#10).
  • No end-of-chapter quick action — the most natural audiobook sleep boundary hides inside the Chapter segment's sub-list.
  • Speed sheet is good — big label, 0.5–3.0 slider step .05 with ticks, presets 0.75/1/1.5/2. Only gap: no fast way home to 1× once you've wandered.
  • Sleep running mode is good — kind label, 68px numeral, ends-at, +5/+10/+15/+30 pills, cancel. Kept whole.

Redesign moves

  • Chapters auto-scroll to current, centered new — plus a search field that appears for books with 50+ chapters. Finished chapters keep the green check (D-STATES: the list shows where you are, not just what exists).
  • Sleep presets arm instantly — tap "30 m" → timer starts, sheet flips to running mode, toast "Sleeping in 30 min", haptic success (bug fix). Start stays for slider/custom/chapter/time values.
  • "End of chapter" pill in the presets row new — one tap arms the chapter-boundary timer without switching segments.
  • "Reset 1×" ghost chip on speed new — appears whenever rate ≠ 1.0.

Screen 03 · Player

Sheets II — queue, sync, recent, notes & the More tray

The "what plays next and what got saved" sheets, plus the More action tray — the sheet behind the action row's More button, which the audit for the full player promised but never drew. The queue is the biggest miss: there is no way to add a book from the player — you back out to the library, long-press, and come back. The recent sheet crams three tap targets into every row. The More tray was a bare icon grid with no state; here it becomes labeled rows that show each action's state and carry the in-context settings + button editor.

Teardown — what's wrong today

  • No add-books path in the queue — Manual and Auto modes accept hand-added rows, but the only entry is the library long-press sheet. Mid-listen queueing means leaving the player entirely.
  • Rule reorder is buried — the Auto rules sub-sheet has toggles only; priority reorder exists solely in the settings editor, two screens away from where you'd notice the order is wrong.
  • Recent rows have 3 crammed targets — row tap resumes at session end, plus two inline timecodes jump to start/end. Mis-taps everywhere on a 54px row.
  • Queue mode segment is good — Off · Manual · Auto · Playlist with captions, pinned now-playing row, bolt icons on rule rows, drag on hand-added rows. Kept whole.
  • Sync sheet is good — plain-language status, last saved, queued offline sessions, Sync-now. Kept; its entry point is the header status indicator (unchanged position, formalized grammar).

Redesign moves

  • "+ Add books" row new — in Manual and Auto modes; opens a search/library picker sheet stacked on the queue. Queueing never leaves the player.
  • Rules sub-sheet gains drag-to-reorder new — priority handles next to the toggles, full parity with the settings editor (D-CONSIST: one editor, not a full one and a crippled one).
  • Recent rows: one primary tap + "…" — row tap resumes at session end (the common case); a small overflow per row holds jump-to-start (D-CONSIST, declutter).
  • Sync indicator grammar formalized — green check-cloud synced / accent pending / red offline; it stays in the header as a status glyph (not an action button) and tap opens this sheet.
  • Notes sheet kept — spoiler-gated thread + composer stamping the live position; entry points unchanged (club cover button, marker dots).
Queue — Auto mode NO CHANGES
One merged up-next list: rule-built rows wear the bolt (read-only), hand-added rows keep drag + remove. New "+ Add books" row sits directly under now-playing.
