Screen 06 · Settings

More tab — the map, redrawn

More is the front door to eighteen settings surfaces. Today the map has thin one-toggle screens, a placeholder Reading screen, and a bare Sync toggle floating outside any group. The redesign keeps every feature but redraws the districts: merged screens where two were thin, promoted screens where depth-3 hid real power, and one deleted screen whose single toggle moves to where the user actually needs it.

Teardown — what's wrong today

  • Search Settings is a whole screen for ONE toggle (searchExternalSources) — a push, a title, a back gesture, for one switch.
  • Haptics is a whole screen for two segments (Feedback Off/Minimal/All + Intensity Light/Medium) — same thin-screen disease.
  • Reading is a placeholder — six disabled rows — while the actual ebook reader ships working prefs (theme/typeface/size/layout in readerPrefs). The settings exist; the screen lies.
  • The Sync toggle is a bare row outside any group (useSharedSettings) — the single most consequential setting on the page, orphaned between Administration and Debug.
  • Queue management is buried at depth 3 (More → Player → Manage queue) even though queue mode, rules and hidden-shelf restores are a daily surface.
  • Diagnostics is a temp device dump with Copy/Refresh sitting in a "Debug" group that reads like scaffolding shipped to users.
  • Admin rows are all "Soon" chips behind a gate that also hides the tab bar — a dead-end hallway (D-NAV violation).
  • Community exposes booleans as Share/Hide segments while every other boolean is a toggle (D-CONSIST).

Redesign moves

  • Search Settings screen deleted — its one toggle moves into unified search's own gear (D-SEARCH). Nothing lost; one less push.
  • Haptics merges into "Appearance & feel" — how the app looks and how it feels are one screen (D-CONSIST).
  • Reading becomes real — mirrors the working reader prefs: theme swatches, typeface, size, layout.
  • Sync gets a proper "Advanced" group with Diagnostics beside it — the orphan row and the Debug group both find a home.
  • Queue promoted to a top-level Playback row — mode, rules and hidden-from-shelves restore in one place, one level down instead of three.
  • Community booleans become toggles — one boolean control everywhere (D-CONSIST).
  • Servers get a visible default star and a "Link a server" row — the long-press affordance is retired.
  • Admin keeps its gate but gets an honest "Coming soon" intro and keeps the tab bar (D-NAV).

Screen 06 · Settings

Appearance & feel

Theme, accent, covers — and now haptics. The current Appearance screen was solid (theme segment, 10-swatch accent picker, cover glow and shape); its only sin was leaving "feel" stranded on a two-control screen next door. Merged, the screen reads as one sentence: this is how HearthShelf looks and feels in your hand.

Teardown — what's wrong today

  • Haptics is its own screen for exactly two segments (Feedback Off/Minimal/All, Intensity Light/Medium) — a push and a back for four seconds of decision.
  • Cover controls are split across three widgets (glow slider 0–60, shape segment, glow style segment) with no live preview of what they do together.
  • Accent swatches have no names on screen — ten dots, no labels, color-blind users guess.

Redesign moves

  • Haptics folds in as the final group — Appearance + Haptics = "Appearance & feel" (D-CONSIST).
  • Live cover preview card above the cover controls — glow, shape and glow style render on a sample cover as you drag.
  • Selected swatch shows its name ("Ember") beside the picker; every swatch gets a 44px touch cell.
  • All existing controls kept verbatim: Theme Auto/Dark/Light/OLED, 10 accents, glow 0–60, Square/Portrait, Gradient/Image.

Screen 06 · Settings

Player settings — one control per value

The player screen is the most-visited settings surface, and today it double-controls its own values: skip forward is BOTH a 15/30/60 chip row AND a 5–300 slider for the same number, twice (forward and back). The redesign collapses each value to one slider with preset tick labels, surfaces the hidden carousel-disables-hotspots interdependency as an inline warning, and makes the whole screen reachable from inside the player as a sheet — settings where you need them.

Teardown — what's wrong today

  • Dual controls for one value, twice — skip forward and skip back each have a chip row (15/30/60) and a slider (5–300) bound to the same setting. Which wins? Both. Confusing.
  • Hidden interdependency — enabling "Swipe between books" silently disables skip hotspots; the only clue is a help-note sentence. Users report "my tap zones broke."
  • Queue lives at depth 3 — mode segment is here but "Manage queue" pushes another screen; rules and restores are two levels from More.
  • No in-context path — changing skip amount mid-listen means leaving the player: More → Player → scroll. (Player-buttons editor already proved the in-context pattern from the player's More sheet.)

Redesign moves

  • One slider per skip value with preset tick labels at 15/30/60 that snap on release — chip row deleted, full 5–300 range kept (D-CONSIST: one control per value).
  • Inline warning row appears when carousel + hotspots are both on, with a one-tap resolution — the interdependency becomes visible the moment it matters.
  • Queue moves out to its own top-level screen (see Editors section); this screen keeps only the end-of-book mode segment with a link.
  • Gear in the player overflow opens these settings as a sheet over the player — change speed or skips without losing your place (frame below).
  • Everything else kept: speed presets, progress-bar scope, background style, tap-artwork, hotspots, carousel, car mode, playlist picker.

Screen 06 · Settings

Sleep timer settings — four features, finally named

The sleep settings screen packs four distinct feature families — rewind, fade, warning beeps, shake-to-extend — plus the auto timer, into four visually identical untitled groups. Nothing tells you where one feature ends and the next begins. The redesign titles every group and keeps every control.

Teardown — what's wrong today

  • Four untitled groups look identical — rewind, fade, beeps and shake read as one undifferentiated wall of toggles and sliders.
  • Auto duration is another dual control — ChipRow + slider bound to one value, the same disease as the skip settings.
  • Beep sub-controls don't indent — CueChips, sound segment, Test button and volume slider sit at the same visual weight as the master toggle.

Redesign moves

  • Titled subgroups: Rewind · Fade out · Warning beeps · Shake to extend · Auto timer — one label per feature family.
  • Auto duration collapses to one slider with preset ticks (matching the skip-slider pattern).
  • Master toggles gate their children — sub-rows dim and collapse when the family is off.
  • All controls kept: rewind 0–120s + stay-in-chapter, fade + length, beep cues (2min/1min/end) + sound + Test + volume, shake per-amount + excessive-shake behavior, quiet-hours time pills.

Downloads & storage

Already one of the healthier screens — auto-download rules, the device storage meter, and live download rows all exist today. The redesign keeps the structure, converts the next-in-queue dual pattern to a chip row alone (it's genuinely discrete: 0/1/3/5), and gives failed downloads a designed retry row instead of a bare error.

Teardown — what's wrong today

  • Failed rows are afterthoughts — a title and an error string, no visible retry affordance (D-STATES).
  • Delete goes through a blocking Alert — heavyweight for a recoverable action; no undo.
  • Max-space slider gives no feedback about what happens at the cap (oldest evicted? downloads blocked?).

Redesign moves

  • Failed rows get an explicit Retry button and a plain-language reason.
  • Delete becomes swipe or button → actionable undo toast — no Alert (matches D-FINISH's toast grammar).
  • Cap behavior stated under the slider: "New downloads pause at the cap."
  • Kept: on-start + continue-listening toggles, next-in-queue chips, StorageMeter bar, allowance, remove-when-finished.

Integrations, sharing & the double-Hardcover fix

Integrations mixes account connections (Hardcover sync, Goodreads import) with display preferences (which external links appear on book pages) — and Hardcover appears in both, so users toggle the link row expecting to disconnect sync. The redesign labels the two jobs as separate subgroups. Sharing's Share/Hide segments become toggles.

Teardown — what's wrong today

  • Hardcover appears twice — once as the sync connection card, once as an external-link toggle — with no framing that these are different things.
  • Community's booleans render as Share/Hide segments while every other boolean in the app is a toggle (D-CONSIST).
  • The Goodreads import flow is good (CSV → match review → confirm) but its entry row gives no hint of the three steps ahead.

Redesign moves

  • Two labeled subgroups: "Connected accounts" (Hardcover card, Goodreads import, server integrations note) and "Links on book pages" (Goodreads / Audible / Hardcover link toggles) — the double-Hardcover reads as two different jobs.
  • Sharing booleans become toggles — reading list, listening now, clubs, club player button, note pops (D-CONSIST).
  • Import row previews the steps — "Pick CSV → review matches → confirm."
  • GoodreadsImportSheet match-review step framed below — the strongest existing flow, kept.

My servers & account

The server list works — cards with icon, name, status, tap-to-switch — but its most important action, setting a default, is a hidden long-press with zero affordance, and there's no way to add a server from here at all. The redesign makes the star a visible button and adds a "Link a server" row.

Teardown — what's wrong today

  • Set-default is long-press only — no hint, no hold indicator; users discover it by accident or never.
  • No add-server action — linking a new server requires signing out or the web flow; the screen that lists servers can't grow the list.
  • Account's profile-photo row is an unimplemented info row presented like a working control.

Redesign moves

  • Visible star button per card — tap the star to set default; filled star marks the current default (long-press retired).
  • "Link a server" row at the foot of the list opens the connect flow.
  • Profile photo row states its truth — "Managed by Gravatar" with the Use-Gravatar toggle beside it; no fake affordance.
  • Kept: switch-on-tap with spinner→check, loading/error+Retry states, account type + member-since, danger sign-out with confirm.

Screen 06 · Settings

The editors — queue & player buttons

Two direct-manipulation editors. The player-buttons editor is today's best settings pattern — a 3-zone drag list reachable both from settings and from the player's More sheet, in context. The queue editor had the same quality but was buried at depth 3; promoted to a top-level Playback screen, it gains the hidden-from-shelves restore list as a visible section.

Teardown — what's wrong today

  • Queue editor at depth 3 (More → Player → Manage queue) for a daily-use surface — rules, manual order and restores all live behind two pushes.
  • Hidden-from-shelves restores hide inside the queue editor — a user looking for "where did my series go" would never look there.
  • Player-buttons editor is genuinely good — 3 zones, drag, quick-hide eye, dual entry — but its settings-side entry gave no preview of the current split.

Redesign moves

  • Queue is a top-level screen — mode segment, drag-to-prioritize Auto rules, and a labeled "Hidden from shelves" restore section, one push from More.
  • Restore rows say what they hide — series vs book, when hidden, one-tap Restore.
  • Buttons editor kept as the in-context reference pattern — its dual entry (settings row + player More sheet) is the model the quick-settings sheet follows.
  • Settings entry rows now preview state: "4 of 6 on screen · 2 in tray".
Queue screen redesigned
Promoted from depth 3. Mode segment, draggable Auto rules, manual items, and the restore list — one surface owns "what plays next".
